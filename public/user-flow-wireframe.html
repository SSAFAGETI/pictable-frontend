메인 런타임 흐름

사용자가 서비스에 진입해 레시피를 탐색하고 커뮤니티 액션까지 이어지는 기본 흐름입니다.

Solid user flow
01 AUTH

로그인 / 회원가입

이메일 또는 Google OAuth로 서비스에 진입합니다.

/login · /signup
02 HOME

홈 탐색

오늘의 추천, 인기 레시피, 최근 레시피를 확인합니다.

/
03 FEED

피드 / 검색

검색어, 태그, 인기순/최신순으로 레시피를 탐색합니다.

/feed
04 DETAIL

레시피 상세

재료, 단계, 좋아요, 저장, 댓글을 확인합니다.

/recipe/:id
05 SOCIAL

저장 / 댓글

좋아요, 저장, 댓글, 답글로 커뮤니티 흐름을 만듭니다.

/recipes/:id/*
06 MY

마이페이지

내 레시피, 저장/좋아요 목록, 구독과 알림을 관리합니다.

/mypage

재료 기반 추천 흐름

찰칵밥상의 핵심 가치인 “가진 재료로 만들 수 있는 요리 추천” 흐름입니다.

Recommendation v2
01 INPUT

재료 입력

사용자가 가진 재료를 직접 입력하거나 이미지로 감지합니다.

/ingredients
02 AI

이미지 재료 감지

업로드 이미지가 Gemini API를 통해 재료 목록으로 변환됩니다.

/media/upload
03 MATCH

추천 계산

ingredient_aliases로 정규화하고 match_rate와 부족 재료를 계산합니다.

/recipes/recommendations
04 RESULT

추천 결과

바로 가능한 레시피와 부족 재료가 적은 레시피를 나누어 보여줍니다.

/recommendations

OAuth 인증 흐름

Google OAuth 콜백이 GitHub나 저장소가 아니라 프론트 라우트로 돌아온 뒤 백엔드 인증 요청으로 이어집니다.

OAuth callback
01 REQUEST

로그인 요청

Auth 화면에서 Google 인증 페이지로 이동합니다.

/login
02 CALLBACK

콜백 / 인증 코드

Google이 authorization code를 프론트 콜백 라우트로 전달합니다.

/oauth/callback
03 EXCHANGE

백엔드 인증 요청

프론트가 code를 백엔드로 보내 JWT 발급을 요청합니다.

POST /auth/google/
04 TOKEN

토큰 저장

발급된 access/refresh token을 Auth State에 저장하고 API 요청에 사용합니다.

Auth State

기획/설계 산출물 → 구현 대조
